**Description**

Prompts 11-13 landed physics in the Nim reference: the format record, the
stateful evaluator, and the state-machine story goldens
`conformance/goldens/m5_physics_story_{rest,excited,settled}.json`. This prompt
makes the Dart runtime honor those goldens, mirroring the Nim implementation
symbol-for-symbol where practical — the same way transform-constraint parity
(bead bony-cz7) mirrored the Nim transform work.

The Dart runtime today has NO physics: `_ConstraintKind` in
`runtime-dart/lib/src/transform.dart:69` is only `{ik, transform, path}`, there
is no `PhysicsConstraintData` in `runtime-dart/lib/src/model.dart`, and no
integrator. Build exactly this:
1. **Integrator.** A Dart port of `runtime-nim/src/bony/constraints/physics_constraints.nim`
   — a new `runtime-dart/lib/src/physics_constraint.dart` mirroring the Nim
   symbols (channels `x/y/rotate/scaleX/shearX`, `PhysicsParams`,
   `PhysicsConstraintState`, `updatePhysicsConstraint`, `integrateChannel`,
   `seedPhysicsChannel`, and the constants `fixedDt=1/60`, `maxFrameDt=0.25`,
   `maxSubsteps=8`, `stepEpsilon=1e-9`, `massEpsilon=1e-6`). All state and
   substep arithmetic in `double`; per-substep operation order is binding and
   must match `docs/physics-integrator-contract.md` and the Nim source exactly.
2. **Model + loader.** `PhysicsConstraintData` in `runtime-dart/lib/src/model.dart`
   (mirror the `TransformConstraintData` class), a `physicsConstraints` list on
   the skeleton model, and JSON + BNB parsing in
   `runtime-dart/lib/src/loader.dart` (mirror the `_parsePath` / transform-
   constraint parse functions). The generated `runtime-dart/lib/src/generated/wire.dart`
   is already regenerated by prompt 11's codegen — do not hand-edit it.
3. **Preserve on rebuild (bug-class guard).** `applyPose` in
   `runtime-dart/lib/src/anim.dart` rebuilds `SkeletonData` (anim.dart:714) and
   must forward `physicsConstraints: data.physicsConstraints` alongside the
   existing `ikConstraints`/`transformConstraints` forwarding (anim.dart:724-728).
   Dropping it silently loses all physics on any posed/animated skeleton — this
   is exactly the constraint-drop defect fixed for IK (bead bony-1c5) and
   guarded for transform (bony-cz7). Add the forward and a regression assertion.
4. **Stateful evaluation seam.** The Dart story path is currently stateless:
   `m5_ik_story_test.dart` does `applyPose(base, evaluated.pose)` then
   `computeWorldTransforms(posed)` per sample (transform.dart), which is fine
   for pure IK/transform/path but NOT for physics. Add a stateful advance seam

   carrying one `PhysicsConstraintState` per constraint, advanced by the
   inter-sample frame `dt`, running a physics stage after the world-transform/
   constraint pass. Extend `_ConstraintKind` (transform.dart:69) and
   `_constraintKindRank` (transform.dart:74) with physics ranked last

   `case _ConstraintKind.path/ik/transform` around lines 887-914: `path` at ~887,
   `ik` at ~897, `transform` at ~906).

   `runtime-dart/test/m5_physics_story_test.dart` mirroring
